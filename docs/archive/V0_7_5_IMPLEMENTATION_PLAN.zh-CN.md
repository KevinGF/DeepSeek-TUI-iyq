# v0.7.5 实施计划

> **English**：[V0_7_5_IMPLEMENTATION_PLAN.md](V0_7_5_IMPLEMENTATION_PLAN.md)

范围：后台 shell 任务中心 TUI 体验、TUI 内 MCP 管理与发现，以及 V4 上下文/缓存策略。本发布线**不包含**提供商扩展或 Whalescale 重命名/迁移工作。

## 上下文/缓存决策

默认路径：

- 保持对话追加式，并保留 DeepSeek V4 缓存复用所需的稳定前缀。
- 默认关闭替换式 `auto_compact`。
- 替换式压缩保持手动或延后：若用户启用 `auto_compact`，V4 仅在接近 80% 模型窗口护栏时压缩（1M 上下文模型为 `800000` token），而非在 reasoning-effort 软顶处。
- Flash 接缝管理器（`[context].enabled`）在 issue #200 有可重复的缓存命中/未命中证据前保持可选。
- 容量控制器默认关闭。除非设置 `capacity.enabled = true`，否则视为遥测或实验性护栏。
- 仅在请求否则会超过模型输入预算时使用紧急溢出恢复。

理由：V4 的 1M token 窗口与前缀缓存经济使早期替换式压缩存疑。可交付的第一片应防止旧 128K 时代启发式在尚无证据表明重写比保留热前缀更便宜可靠之前改写上下文。

## 可交付切片

### 切片 1：上下文策略与文档

- 将默认 `auto_compact` 改为关闭。
- 保持 V4 替换压缩阈值延后且与 reasoning effort 独立。
- `[context].enabled` 默认为 false。
- 使 `docs/CONFIGURATION.md`、`docs/capacity_controller.md` 与 `config.example.toml` 与代码默认一致。
- 为默认与 V4 阈值行为增加针对性测试。

### 切片 2：后台 shell 任务中心（#195）

- 增加由 `ShellManager::list()` 驱动的任务中心视图。
- 展示命令、cwd、关联任务 id（若有）、状态、已用时间、退出码与最新输出。
- 增加控件：查看完整输出、轮询最新输出、对支持 PTY/stdin 的任务发送 stdin、终止后台任务、将已完成输出挂为任务证据。
- 对重启后过期的任务明确标记为陈旧，而非当作仍在运行。
- 增加生命周期测试：启动、轮询、取消、完成、陈旧/重启，以及运行中与已完成任务详情的 TUI 快照。

### 切片 3：MCP 管理器（#196）

- 增加 `/mcp` 或命令面板动作以打开 MCP 管理视图。
- 展示解析配置路径、服务器启用/禁用、传输、命令/URL、超时、启动错误，以及发现的工具/资源/提示数量。
- 将 `mcp_config_path` 接入交互配置界面。
- 支持 init、添加 stdio 服务器、添加 HTTP/SSE 服务器、启用、禁用、移除、验证、重连，以及检查工具/资源/提示。
- 同时保留 `servers` 与 `mcpServers` 配置形状。

### 切片 4：MCP 可发现性（#197）

- 增加由与管理器相同发现状态支撑的 MCP 命令面板分区。
- 按服务器分组工具/资源/提示。
- 禁用/失败服务器不阻塞面板渲染。
- 与 `mcp_<server>_<tool>` 保持模型可见命名一致。

## 停止规则

- 除非已验证的 PR 实际解决，否则不要关闭 #159 或 #162。
- 不要增加提供商扩展。
- 不要重命名或迁移到 Whalescale。
- 不要将 TUI 扩成大型重设计；每个切片应可独立测试与发布。
